KATA PENGANTAR

Puji syukur penulis panjatkan atas selesainya penyusunan dokumen "User Guide Application" atau Panduan Penggunaan Aplikasi untuk Sistem Informasi Barang Habis Pakai (SIBHP). Dokumen ini disusun untuk melengkapi kriteria penilaian N-8 (Membuat User Guide Application) dalam pelaksanaan Uji Kompetensi Keahlian (UKK).

Aplikasi SIBHP ini dirancang untuk memudahkan manajemen pendataan, pengajuan permintaan, hingga pelaporan barang stok di lingkungan instansi/perusahaan secara digital dan terstruktur. Tersedia dua hak akses utama dalam aplikasi ini, yaitu hak akses Admin Gudang dan hak akses Pegawai (User).

Semoga panduan singkat ini dapat memberikan petunjuk yang jelas demi kelancaran operasional SIBHP.

BAB I
PANDUAN PENGGUNA (PEGAWAI)

Bagian ini menjelaskan langkah-langkah penggunaan aplikasi bagi peran Pegawai atau Staf biasa yang ingin melakukan permohonan pengambilan barang habis pakai.

1.1. Login Pegawai

Langkah pertama untuk menggunakan aplikasi adalah melakukan proses otentikasi (Login).

  1. Buka aplikasi SIBHP melalui browser.
  2. Masukkan Email/Username dan Password akun pegawai Anda.
  3. Klik tombol Masuk / Sign In. Jika berhasil, Anda akan diarahkan ke halaman Dashboard (Beranda).
📷 [AMBIL SCREENSHOT HALAMAN TAMPILAN AWAL / LOGIN DI SINI]

1.2. Halaman Dashboard Pegawai

Setelah melakukan login, pegawai akan disambut oleh halaman Dashboard. Halaman ini berisi informasi ringkas mengenai total jumlah barang yang pernah Anda minta, status persetujuan, dan akses pintas untuk membuat permintaan baru.

📷 [AMBIL SCREENSHOT HALAMAN DASHBOARD SEBAGAI USER]

1.3. Membuat Permintaan Barang Baru

Prosedur inti dari sisi pengguna adalah mengajukan pengadaan/pengambilan barang kepada bagian gudang.

  1. Pilih menu Permintaan Barang -> Buat Permintaan Baru.
  2. Isi form utama: Prioritas Permintaan (Biasa/Segera), Tanggal Dibutuhkan, Divisi, dan Catatan khusus bila ada.
  3. Klik tombol Tambah Barang, lalu pilih barang apa saja yang ingin Anda ambil, beserta jumlah dan rincian keperluannya.
  4. Jika tabel daftar barang yang Anda isi sudah benar, klik tombol Kirim Surat Permintaan.
📷 [AMBIL SCREENSHOT HALAMAN FORM INPUT PERMINTAAN BARU]

1.4. Melihat Riwayat & Status Permintaan

Pegawai dapat melacak sejauh mana permohonan mereka diproses oleh admin, dan juga memanfaatkan kotak pencarian (search) untuk menemukan nomor dokumen spesifik.

  1. Buka menu Permintaan Barang -> Riwayat Permintaan.
  2. Anda akan melihat daftar surat yang pernah Anda buat beserta statusnya, seperti "Menunggu Admin", "Disetujui", "Ditolak", atau "Selesai".
  3. Anda juga dapat mengetik kata kunci pada kotak pencarian untuk mencari dokumen tertentu.
  4. Klik tombol aksi Detail (ikon mata) untuk melihat rincian barang apa saja yang disetujui beserta catatan dari Admin Gudang.
📷 [AMBIL SCREENSHOT HALAMAN TABEL RIWAYAT PERMINTAAN USER (yang menampilkan fitur search bar yang kita tambahkan tadi)]

BAB II
PANDUAN ADMINISTRATOR GUDANG

Bagian ini dikhususkan bagi staf administrator gudang yang memiliki wewenang penuh dalam mengelola lalu lintas barang, transaksi, persetujuan, hingga pelaporan (Kartu Persediaan).

2.1. Memproses Permintaan dari Pegawai

Setiap surat permohonan yang diajukan oleh pegawai harus diproses (Disetujui atau Ditolak) oleh Admin.

  1. Login sebagai Admin, lalu buka menu Permintaan Barang -> Menunggu Persetujuan.
  2. Klik tombol Proses Persetujuan pada daftar permintaan yang masuk.
  3. Sebuah jendela Pop-Up (Modal) akan muncul. Admin dapat mengatur dan memodifikasi angka pada kolom Jumlah Disetujui (apabila barang tidak mencukupi, admin berhak menyetujui setengahnya saja). Berikan catatan jika perlu.
  4. Jika ingin menolak sepenuhnya, klik tombol Tolak dan sertakan alasan tertulis.
📷 [AMBIL SCREENSHOT HALAMAN MENUNGGU PERSETUJUAN (beserta Pop Up Modal Persetujuannya)]

2.2. Manajemen Master Barang

Katalog inventaris dikelola terpusat di halaman ini.

  1. Akses menu Master Data -> Master Barang.
  2. Di sinilah Admin dapat melakukan Tambah Barang Baru (Create), Edit Barang, maupun pengecekan saldo stok minimum.
  3. Fitur Import Data juga tersedia di sini. Admin dapat mengunduh Template Excel, mendatanya secara *offline*, lalu mengunggah *file CSV*-nya kembali agar ribuan barang tersimpan massal secara otomatis.
📷 [AMBIL SCREENSHOT HALAMAN MASTER BARANG (Tampilkan juga jika ada form import file CSV-nya)]

2.3. Transaksi Barang Masuk

Apabila stok di gudang mulai menipis, pengadaan barang baru maupun logistik dari *Supplier* harus tercatat di sistem.

  1. Pilih Menu Transaksi -> Barang Masuk.
  2. Klik Tambah Data, dan masukkan rincian Nomor Dokumen Surat Jalan/NUSP, Nama Supplier, lalu ketik nama atau kode barang pada *combo box* untuk memilih katalog barang.
  3. Masukkan spesifikasi dan kuantitas barang masuk. Secara sistem, jumlah ini akan menambah stok akhir otomatis.
  4. Pada halaman riwayat barang masuk, Anda pun dapat melakukan Filter Bulan/Tanggal dan Pencarian nomor surat sekaligus menggunakan satu tombol penelusuran.
📷 [AMBIL SCREENSHOT HALAMAN BARANG MASUK (Tabel riwayat dengan filter/search barna)]

2.4. Pencetakan Kartu Persediaan Barang

Kartu persediaan adalah hal krusial untuk kegiatan *stock opname* dan akuntansi logistik.

  1. Buka menu Laporan -> Kartu Persediaan.
  2. Layar pertama akan menampilkan halaman Manajemen Barang untuk memantau stok total saat ini, total barang habis, dan stok menipis. Lengkap dengan kotak pencarian terpusat.
  3. Untuk melihat Buku Besar Persediaan *(Stock Ledger)* dari 1 item, klik ikon (Riwayat) pada baris barang tersebut.
  4. Layar "Riwayat Stok (Kartu Persediaan)" akan terbuka, menampilkan aliran debet-kredit jumlah barang yang bertambah dan berkurang berdasarkan tanggal transaksinya. Anda bisa menekan tombol **Cetak Laporan** (PDF) untuk di-print sebagai bukti fisik.

PENUTUP

Demikianlah Panduan Penggunaan Sistem Informasi Barang Habis Pakai (SIBHP) ini dibuat. SIBHP diharapkan mampu mengakomodir alur pergerakan barang mulai dari gudang masuk hingga didistribusikan ke masing-masing divisi agar tertata rapi secara administratif.

Seluruh fitur mulai instalasi hingga operasi perancangan tampilan di atas telah mengikuti prinsip logika rekayasa perangkat lunak.
